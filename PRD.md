# Product Requirements Document (PRD) - Learn Quran Offline Mobile App

## 1. Executive Summary
The **Learn Quran Offline Mobile App** is a privacy-first, fully offline-capable mobile application built using **Flutter**. The app aims to help users learn, understand, and connect with the Holy Quran in their native languages. 
Beyond simple rote memorization, the app focuses on deep understanding and provides a respectful, gentle, and calm Q&A experience modeled after the Sunnah of Prophet Muhammad (SAW). It uses local LLM execution (Gemma 4 e2b and e4b) coupled with a local RAG (Retrieval-Augmented Generation) pipeline to ensure high factual accuracy, verifiable sources, and zero fabrications.

---

## 2. Core Vision & Design Philosophy
*   **Sunnah-Inspired Q&A Interaction:** Answering user questions must always be calm, gentle, and respectful, adopting the teaching methodology of Prophet Muhammad (SAW) (gentle, encouraging, and clear).
*   **Factual Rigor (Anti-Fabrication):** The app must restrict its answers strictly to verified sources (Quran, Sahih Hadiths, approved scholarly commentaries). If a question is outside the scope or source bounds, the app should gently decline to answer rather than risk fabricating information.
*   **Offline-First & Privacy-First:** All search, vector database retrieval, and model inference run locally on the user's device. No personal data, search queries, or conversation logs are uploaded to any server.

---

## 3. Key Feature Specifications

### 3.1. Interactive & Understanding-Focused Quran Reader
*   **Original Arabic Text:** High-quality Arabic typesetting with word-by-word syntax, grammar breakdown, and audio recitation.
*   **Multilingual Support:** Default translations in **English** (e.g., Sahih International / Clear Quran), **Bangla**, and **Hindi**.
*   **Memorization & Tafsir Companion:** Interactive tools to highlight verses, listen to repetitions, and view verified Tafsir (scholarly commentaries).

### 3.2. Sunnah-Style Q&A Agent & RAG Pipeline
*   **Local Inference:** Runs **Gemma 4 e2b** (default for lower-end devices) or **Gemma 4 e4b** (default for higher-end devices) locally. The UI will include a selector to allow manual overriding.
*   **Robust Local RAG:** Utilizes a local vector store (e.g., SQLite with a custom vector module or local ONNX embeddings) containing:
    *   The Holy Quran (Arabic and target translations).
    *   Authentic Hadith collections (Sahih al-Bukhari, Sahih Muslim, Riyadh as-Salihin, etc.).
    *   Scholarly commentaries (Tafsir Ibn Kathir, etc.).
    *   Reference material from other religions (specifically for addressing comparative theological questions respectfully).
*   **Source Citation:** Every answer generated by the agent must cite its precise verse (Surah:Ayah) or Hadith number.

### 3.3. Salat (Prayer) Reminders
*   **Offline Calculation:** Local calculation of prayer times using GPS coordinates or manual location input.
*   **Gentle Notifications:** Non-intrusive, gentle reminders for Salat (Athan or peaceful reminders), keeping the user's current context in mind.

### 3.4. Dynamic User-Centric Experience (Curated Content)
*   **Engagement Tracking:** An offline, privacy-preserving state tracking mechanism that notes the user's current progress, reading frequency, and interests.
*   **Daily Curated Story:** A local engine compiles a daily story or moral lesson based on the user's recent interactions, readings, or questions, providing customized, relevant reflections.
*   **Engaging Conversations:** Local agents initiate check-ins and prompts to encourage reflection and daily reading.

## 4. Non-Functional Requirements & Constraints

| Attribute | Specification |
|---|---|
| **Platform** | Cross-platform iOS and Android via Flutter (Dart) |
| **Inference Models** | Gemma 4 e2b (low-end devices), Gemma 4 e4b (high-end devices) |
| **Execution** | 100% offline (no active internet required for core operation) |
| **Localization** | Arabic, English, Bangla, Hindi |
| **Memory/Storage** | The local vector DB and model files must be highly compressed to run on mobile storage budgets. |
| **Source Validation** | Strict deterministic check on RAG outputs to prevent hallucinations. |

---

## 5. Scope Boundaries (Nice-to-Have vs. Must-Have)
*   **Must-Have (MVP):**
    *   Full offline Quran reader (Arabic, English, Bangla, Hindi).
    *   Offline vector search on Quran & Hadith.
    *   Local Gemma 4 e2b/e4b model integration for gentle Q&A.
    *   Offline Salat calculation and notification.
*   **Nice-to-Have (Future Releases):**
    *   Audio recitation offline downloads (due to large file size).
    *   Interactive gamified quizzes/memorization challenges.
